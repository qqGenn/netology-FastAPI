# Task Manager API - Документация проекта

## Краткое описание

Task Manager API — это RESTful API для управления задачами, реализованный на FastAPI. Проект предоставляет эндпоинты для создания и получения данных задач.

## Структура проекта

```
Netology/FastAPI/
├── main.py           # Основной файл приложения с FastAPI-эндпоинтами
├── models.py         # Pydantic-модели для валидации данных
├── requirements.txt  # Зависимости проекта
├── docs/             # OpenAPI спецификация
│   └── openapi.json
```

## Описание файлов

### main.py
Основной файл приложения, содержащий:
- Настройку FastAPI приложения с метаданными
- Временное хранилище задач в памяти (`tasks: list[TaskResponse]`)
- Эндпоинты:
  - `GET /` — корневой endpoint для проверки работоспособности
  - `POST /tasks` — создание новой задачи
  - `GET /tasks/{task_id}` — получение задачи по ID

### models.py
Pydantic-модели для валидации данных:
- `TaskCreate` — модель входных данных для создания задачи
- `TaskResponse` — модель ответа с данными задачи

## Модели Pydantic

### TaskCreate (входные данные)
| Поле | Тип | Обязательное | Описание | Валидация |
|------|-----|--------------|----------|-----------|
| title | string | Да | Название задачи | min_length=3, max_length=100 |
| description | string | Нет | Описание задачи | — |
| priority | int | Да | Приоритет задачи | ge=1, le=5 |

### TaskResponse (ответ)
| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Уникальный идентификатор задачи |
| title | string | Название задачи |
| description | string \| null | Описание задачи |
| priority | int | Приоритет задачи (1-5) |
| created_at | datetime | Дата создания в формате ISO 8601 |

## Инструкция по запуску

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите сервер:
```bash
py -m fastapi dev main.py
или
uvicorn main:app --reload
```

3. Откройте браузер и перейдите по адресу:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## OpenAPI документация

- OpenAPI спецификация: `docs/openapi.json`
- Swagger UI: http://localhost:8000/docs
- Спецификация API: http://localhost:8000/docs/openapi.json

## Возможные ограничения

1. **Временное хранилище** — данные сохраняются только в памяти и теряются при перезапуске сервера
2. **Отсутствие аутентификации** — эндпоинты не защищены
3. **Отсутствие базы данных** — нет постоянного хранения данных
4. **Отсутствие обновления/удаления** — не реализованы PUT/PATCH/DELETE операции
5. **Отсутствие пагинации** — нет возможности получения списка задач

## Эндпоинты API

### POST /tasks
Создает новую задачу.

**Тело запроса:**
```json
{
  "title": "Сходить в кино",
  "description": "Выбрать интересный фильм, купить билеты в последний ряд, взять прохладительные напитки и наслаждаться",
  "priority": 3
}
```

**Ответ 201:**
```json
{
  "id": 1,
  "title": "Сходить в кино",
  "description": "Выбрать интересный фильм, купить билеты в последний ряд, взять прохладительные напитки и наслаждаться",
  "priority": 3,
  "created_at": "2026-07-09T12:00:00.000000Z"
}
```

**Ошибки:**
- 422 — Неверные данные валидации

### GET /tasks/{task_id}
Получает задачу по ID.

**Ответ 200:**
```json
{
  "id": 1,
  "title": "Сходить в кино",
  "description": "Выбрать интересный фильм, купить билеты в последний ряд, взять прохладительные напитки и наслаждаться",
  "priority": 3,
  "created_at": "2026-07-09T12:00:00.000000Z"
}
```

**Ошибки:**
- 404 — Задача не найдена
- 422 — Неверный формат task_id
